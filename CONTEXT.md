# SwitchHack Hub — Build Context

## Project
switchhack.com — a Nintendo Switch homebrew community hub hosted on GitHub Pages.
Local dev → push to GitHub → auto-deploys.
Stack: vanilla HTML/CSS/JS, Tailwind CSS (local copy at assets/css/tailwind.min.css),
custom styles at assets/css/styles.css, shared nav via assets/js/nav.js.

## Hub Page Purpose
The hub (hub.html) serves experienced Switch modders who want to answer:
1. Is my setup still good? (firmware/Atmosphere/Hekate status)
2. Did anything important release recently? (GitHub release feed)
3. Is anything happening in the scene worth knowing? (scraper pulse)
4. Where do I go for X? (resource directory)

## Page Architecture — 4 Zones

**Zone 1 — Status Bar** (full width, slim strip at top)
Shows: Firmware version, Atmosphere version, Hekate version, last sync time, overall status chip.
Data: Atmosphere + Hekate from GitHub API. Firmware from monitor_data.json.
Cached in localStorage for 1 hour.

**Zone 2 — Release Feed** (left column, ~58% width on desktop)
Shows: Latest GitHub releases from tracked repos, grouped by category.
Categories: Core CFW, Overlays, Sysmodules, File & Save Tools, Homebrew Apps, Emulation.
Filter pills let user isolate a category.
Only shows repos with releases in last 90 days.
Data: GitHub Releases API, 1hr localStorage cache.

**Zone 3 — Scene Pulse** (right column, ~42% width on desktop)
Shows: Warnings/alerts (hidden if none), top Reddit/GBAtemp posts.
Data: monitor_data.json (scraper pipeline). If missing, zone collapses gracefully.
Zones 1 and 2 are completely independent of this — scraper failure doesn't break them.

**Zone 4 — Resource Directory** (full width, below fold)
Shows: Communities, Essential Guides, Core Repositories by category, Also Worth Knowing.
Data: Mostly static HTML. Repo cards rendered from REPOS config in hub.js.

## Important Notes for Future Sessions
- Never modify nav.js
- Always test GitHub API calls work before pushing (check browser console)
- The scraper/monitor pipeline is a separate backend concern — hub.js only reads monitor_data.json
- GitHub unauthenticated API rate limit: 60 requests/hour per IP. With 1hr cache this is fine.
- If rate limit becomes an issue: upgrade to Cloudflare Worker proxy (free tier)

## GitHub API Architecture
All GitHub API calls are proxied through a Cloudflare Worker at:
switchhack-hub.lunt34.workers.dev

The Worker injects a GitHub PAT token server-side (stored in Worker env vars).
This gives 5000 req/hr instead of 60.
hub.js sets GH_API to the Worker URL — never change it back to api.github.com directly.
If the Worker is down, fall back behavior uses stale localStorage cache.
